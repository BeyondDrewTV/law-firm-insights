# experiments.md
# Clarion — Experiment Log
# Version: 1.0

---

## Purpose

This file tracks active, completed, and proposed experiments across the company.
Experiments are how Clarion tests hypotheses before committing to a strategy.

Agents may propose experiments in their reports using the PROPOSED EXPERIMENT block.
The CEO reviews proposals and adds approved experiments to the ACTIVE EXPERIMENTS
section below. Only the CEO (or authorized operator) may add, modify, or archive entries.

Agents read this file to stay aware of what is being tested. Agents never modify it.

---

## How an Experiment Moves Through the System

1. Agent files a PROPOSED EXPERIMENT block in its weekly report.
2. Chief of Staff surfaces the proposal in the CEO brief under ACTIVE EXPERIMENTS.
3. CEO reviews and either adds it here as Active or declines it.
4. Once a success metric is evaluated, CEO moves the experiment to COMPLETED.
