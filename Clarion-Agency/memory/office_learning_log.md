# office_learning_log.md
# Clarion — Office Learning Log
# Version: 1.0
#
# PURPOSE
# This file is the single approved ledger for improvements to the agent office's
# memory files, operating rules, and agent prompts.
#
# HOW IT WORKS
# 1. Agents may include a LEARNING PROPOSAL block in any report.
# 2. The Chief of Staff collects all proposals and surfaces them in the
#    CEO brief under OFFICE LEARNING.
# 3. The CEO reviews proposals and approves, rejects, or defers each one.
# 4. Approved proposals are applied manually to the target memory file.
# 5. Every decision — approved, rejected, or deferred — is logged here.
#
# GUARDRAILS
# - No agent may modify this file or any other memory file directly.
# - No runner or workflow may write to memory/ at runtime.
# - Only the CEO (human) may apply approved changes.
# - Rejected proposals are logged here and not revisited unless new evidence
#   surfaces.
#
# LOG FORMAT
# Each entry:
#   Date:     YYYY-MM-DD
#   Agent:    [agent name]
#   Target:   [memory file the proposal would modify]
#   Proposal: [what the agent proposed, verbatim or summarized]
#   Decision: [Approved | Rejected | Deferred]
#   Rationale:[Why. Required for Rejected and Deferred.]
#   Applied:  [YYYY-MM-DD | Pending | N/A]
#
