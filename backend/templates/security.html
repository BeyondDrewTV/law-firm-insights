Security

Practical safeguards for client feedback data.

Security is built into the product workflow so partners and operations teams can use reporting confidently.

Data protection

HTTPS in transit, secured sessions, and account-scoped data separation.

Access and controls

Authenticated login, password hashing, and rate-limited access endpoints.

Responsible use

No ad resale and clear options to export or remove reports from your account.

Data protection

  • HTTPS protects data in transit in production.
  • Session cookies use HttpOnly and SameSite settings.
  • Secure cookie flag is enabled in production.

Access and controls

  • Passwords are hashed and never stored in plain text.
  • Rate limits help reduce brute-force and abuse attempts.
  • Account access is scoped to the authenticated user context.

Responsible use

  • Customer data is not sold for advertising.
  • Operational logs avoid sensitive values like passwords.
  • Users can remove uploaded reports from the app.

Issue Reporting

Need to report a security concern?

Contact channel

Email security@clarionhq.co with details and reproduction steps.

Response handling

Production errors return generic messages while internal details are reviewed server-side.
