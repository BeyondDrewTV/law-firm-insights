# Clarion Governance Design System v1.0

This system defines governance interface rules, not visual decoration.

## Core Standard
- Signal over decoration.
- Every UI element must clarify risk, execution, or accountability.

## Visual Tone
- Institutional, calm, structured, stable, high-trust.
- Avoid startup dashboard styling, decorative effects, and playful motion.
- No gradients, glow, or heavy shadows in governance surfaces.

## Layout Rules
- 8px spacing rhythm only: 8, 16, 24, 32, 48, 64.
- Three container levels:
1. Executive Surface (one per page)
2. Section Container
3. Inline Modules

## Typography Rules
- Hierarchy by size and weight, not color.
- Eyebrow labels: uppercase micro text.
- H1: clear and strong.
- H2: medium weight section structure.
- Metric number: largest type, used only for executive risk metrics and critical counters.

## Color Rules
- Neutral base for backgrounds, surfaces, borders, and body text.
- Semantic risk scale only:
1. Controlled (muted green)
2. Watch (muted amber)
3. Elevated (muted orange)
4. Critical (muted red)
- Use risk colors only for badges, thin borders, indicators, and progress fills.

## Buttons
- Exactly three tiers:
1. Primary (global action only)
2. Secondary (supporting actions)
3. Text Link (contextual navigation)
- Do not place competing solid primary buttons together.

## Tables
- Uppercase micro headers.
- Consistent row height.
- Subtle separators.
- Right-aligned action column.
- Badge styling must stay subdued and consistent.

## Badges
- Small, muted, and consistent.
- Use subtle fills and borders only.
- No high-saturation fills or oversized treatments.

## Empty States
- Must be explicit and informative.
- No illustration or playful tone.

## Motion
- Minimal only: hover, focus, subtle expand/collapse, smooth transitions.
- No animated counters, bounce, pop, or hero slide effects.

## Density + Hierarchy
- Moderate density: breathable but efficient.
- Top 30% of page must feel heavier than lower sections.
- If lower modules compete with executive surface, hierarchy is broken.

## Anti-Drift Check
Before adding any new card, chart, color accent, motion, or icon:
- Ask whether it clarifies governance posture.
- Remove it if it does not.
